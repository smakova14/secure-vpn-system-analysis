# Подробная спецификация требований (SRS)

## 1. Анализ регуляторов и нормативной документации
В рамках проектирования системы были проанализированы следующие документы:
* **ФЗ-152 «О персональных данных»:** Система обязана обеспечивать конфиденциальность логинов и метаданных сессий. Принято решение об использовании TLS 1.3.
* **Приказ ФСТЭК №21:** Внедрена мера УЗД.1 (Идентификация и аутентификация субъектов доступа).
* **Стандарт СТО БР ИББС (для финтеха):** Требование двухфакторной аутентификации для доступа к критическим узлам сети.

## 2. Матрица трассировки (Требования vs Реализация)

| ID | Бизнес-требование | Функциональное требование | Метод реализации |
|:---|:---|:---|:---|
| 01 | Защита от подбора пароля | Блокировка учетной записи | Brute-force protection на уровне шлюза |
| 02 | Удаленный доступ | Шифрование трафика | IPsec / SSL VPN туннель |
| 03 | Контроль доступа | Ролевая модель (RBAC) | Интеграция с группами Active Directory |

## 3. Сетевой стек (Deep Dive)
Для вакансии важно понимание TCP/IP:
* **L3:** IP-адресация внутри туннеля (Virtual IP pool).
* **L4:** Контроль целостности через TCP checksum.
* **Security:** Механизм Perfect Forward Secrecy (PFS) для того, чтобы компрометация одного ключа не позволила расшифровать прошлые сессии.
